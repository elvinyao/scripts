下面给出一个示例流程，展示如何使用 git filter-branch 来“排除”指定目录，让它们在历史中彻底消失，并将修改后的历史（包括原有的提交、tags 等）推送到另一个远程仓库。请根据实际需求调整命令中的路径、分支、远程地址等。

--------------------------------------------------------------------------------
1. 准备工作
--------------------------------------------------------------------------------
① 确保你已经在本地有一份完整的仓库（包含所有分支和tag），并且工作目录干净（无未提交的更改）。
② 为了安全起见，最好先做一个备份分支或把整个项目目录复制一份留作备份。

--------------------------------------------------------------------------------
2. 移除指定目录并重写历史
--------------------------------------------------------------------------------
下面的示例命令将会从历史中移除 “dir_to_remove1” 和 “dir_to_remove2” 两个目录（假设它们位于项目根目录）。你可以一次性写多个目录，用空格分隔：

git filter-branch --index-filter '
    git rm -rf --cached --ignore-unmatch dir_to_remove1
    git rm -rf --cached --ignore-unmatch dir_to_remove2
' \
--tag-name-filter cat \
--prune-empty \
-- --all

说明：  
• --index-filter "…" ：在每一个提交上执行一次指定脚本，这里通过 git rm 移除目录。  
• --ignore-unmatch ：即使某些提交里没有这几个文件/目录，也不会报错。  
• --tag-name-filter cat ：保持 tag 名称不变。  
• --prune-empty ：如果有的提交在移除目录后变成了空提交，会被自动丢弃。  
• -- --all ：对本地仓库中所有分支、标签等引用进行历史重写。

执行命令后，git 会为每个commit记录执行一次“移除指定目录”的操作，并重写整个历史。完成后你可以检查一下历史，比如查看某些老提交，确认那几个目录已经不在对应提交中了。

--------------------------------------------------------------------------------
3. 清理并重新打包
--------------------------------------------------------------------------------
为了让仓库体积变小，同时避免旧对象留在 reflog 中，可以执行：

rm -rf .git/refs/original   # 删除 filter-branch 产生的 original 引用  
git reflog expire --expire=now --all  
git gc --prune=now --aggressive

这样可以确保被删除的目录及相关对象真正被 Git 垃圾回收（GC），不会继续占用空间。

--------------------------------------------------------------------------------
4. 修改远程仓库地址并推送
--------------------------------------------------------------------------------
① 先把本地原先的 origin 远程地址指向新的地址（假设新的远程仓库地址为 https://example.com/new/repo.git）：

git remote set-url origin https://example.com/new/repo.git

② 将所有分支和标签推送到新的远程仓库（由于重写了历史，需要使用 --force，否则会被拒绝）：

git push --force --all
git push --force --tags

--------------------------------------------------------------------------------
5. 验证
--------------------------------------------------------------------------------
• 在新的远程仓库中，检查对应分支、标签，确认那几个不需要的目录已经不存在。  
• 若原仓库依旧保留，需要注意该仓库与新仓库现在的提交哈希值已经不一致，二者完全分离。今后如果需要合并或拉取到一起，会比较麻烦，务必确认自己的需求。

--------------------------------------------------------------------------------
小结
--------------------------------------------------------------------------------
以上流程中最关键的部分是使用 git filter-branch --index-filter 命令来对每一个提交进行“目录移除”的操作，并借助 --prune-empty 丢弃空提交。最后，用 --force 把重写历史后的分支与标签推送到新的远程仓库。

如果只需要保留特定目录（而非移除目录），也可以考虑先在本地新建一个空仓库，然后用 git subtree 或者 git filter-branch 方式，反向筛选出需要的目录，效果与上述流程类似，只是换了保留/移除的角度。  

至此，你就能在新的仓库中只保留所需的目录及其完整提交历史（包含所有旧的 commits 和 tags），而不再包含被排除的目录内容。祝一切顺利!
